Where we'd like to go with Lucid

Login/Admin

One of the first things we'd like to do is implement a log-in or admin feature so that some form of authorization is required for editing or submitting.

Difficulty: Easy (login), Hard (admin)

Robust functionality

We'd like to make the site more robust against bad entries, for example, blank entries in fields.

Difficulty: Easy

Plug-in

We want to create a browser plug-in for Lucid that detects the url of the user's current page. If an entry for the url exists in Lucid, the plugin should link to that entry. Otherwise, the plugin should link to the Lucid homepage.

Difficulty: Moderate

Categorization

We'd like to be able to tag companies according to their function (for example, "search engine" or "social network") so that users can filter the list of catalogued privacy policies according to function of service.

Difficulty: Moderate

Modularity, abstraction

Lucid is currently built on top of Flask, a Python microframework, with SQLite. Our goal is to switch to a more high-level application framework that will be more flexible with respect to database management and structural changes.

Difficulty: Moderate

Resources

We'd love to make Lucid an educational resource for privacy in general. We could build a privacy-related blog coupled with a resources/introduction page.

Difficulty: Moderate

Third parties

We'd like to create profiles for third parties that collect information from consumers. A privacy page for a company would ideally link to all of its third parties, and a profile for a third party would link back to all the companies associated with that third party. Although some companies disclose all third parties, others do not; this could potentially be dealt with by mining data from applications like Collusion.

Difficulty: Hard
